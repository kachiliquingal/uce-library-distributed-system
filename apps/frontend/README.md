# Frontend Service

## Overview
The Frontend Service is the user-facing web application for the UCE Library Distributed System. It provides a seamless, responsive, and highly interactive user interface for library patrons and administrators. Built with modern web technologies, it ensures high performance, elegant UI/UX, and maintainability.

## Responsibilities
- **User Interface (UI):** Render the visual components for authentication, catalog browsing, and user management.
- **State Management:** Handle client-side state using lightweight global stores (Zustand).
- **API Integration:** Communicate securely with distributed backend microservices (Auth and Catalog).
- **Routing:** Provide client-side routing for seamless single-page application (SPA) navigation.

## Infrastructure
- **Core Framework:** React 18
- **Build Tool:** Vite (for Lightning-fast HMR and optimized production builds)
- **Styling:** TailwindCSS v4 with modern design aesthetics
- **Containerization:** Deployed via Docker, served by a lightweight Nginx web server.
- **Port:** 5173 (Development) / 80 (Production Container)

## Development Setup

To run the frontend service independently:

1. Navigate to the frontend directory:
   ```bash
   cd apps/frontend
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Environment Variables:
   Create a `.env.local` file for local development:
   ```env
   VITE_AUTH_API_URL=http://localhost:3001/api/auth
   VITE_CATALOG_API_URL=http://localhost:3002/api/catalog
   ```

4. Start the development server:
   ```bash
   npm run dev
   ```

## Production Build

The production build is fully automated via the CI/CD pipeline. The GitHub Actions workflow dynamically injects the `VITE_AUTH_API_URL` and `VITE_CATALOG_API_URL` based on the real-time IP addresses generated by Terraform before building the Docker image.
